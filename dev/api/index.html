AssemblyLine2PC.raw_materials — Constant
raw_materials

String ordered vector of raw resources that anchor all cost tuples produced by AssemblyLine2PC.jl.

source
AssemblyLine2PC.build_skeletontree — Method
build_skeletontree()

Create an empty recipe graph seeded with vertices for every known item. Edges only store ingredient ratios; raw-resource footprints are populated later by datatree.

source
AssemblyLine2PC.cost — Method
cost(graph, item)

Return the total raw-material tuple required to produce a single unit of item.

The tuple ordering matches raw_materials so each entry represents the per-unit demand for the corresponding raw resource. This helper is convenient when you already have a pre-computed graph from datatree and need to inspect multiple items without re-running the traversal.

source
AssemblyLine2PC.datatree — Method
datatree()

Construct the complete crafting graph for Assembly Line 2.

The resulting MetaGraph stores per-item resource footprints as vertex metadata so downstream analysis can reuse the same structure without re-traversing recipe data. The tuple stored on each vertex aligns with raw_materials, ensuring deterministic indexing for raw material accounting and visualization routines.

source
AssemblyLine2PC.nminers — Function
nminers(item, rate=1, data=datatree())

Estimate how many Miners are required to maintain rate units per second of item.

Each Miner extracts 5 units of raw material per second in Assembly Line 2. The function uses total_material to compute the throughput requirements and then converts that to the number of working Miners needed on the resource patches.

source
AssemblyLine2PC.topspeed — Function
topspeed(item, miners, data=datatree())

Compute the theoretical maximum units per second of item that miners active Miners can support.

This is the inverse of nminers and is useful when you know the amount of extraction capacity available and want to determine the production ceiling before optimizing factory layout details.

source
AssemblyLine2PC.total_material — Function
total_material(item, rate, data=datatree())

Calculate the total raw-material throughput required to sustain rate units per second of item.

This multiplies cost by the desired production rate so downstream sizing and balancing can be performed in a single step.

source
AssemblyLine2PC.vertex_costs — Function
vertex_costs(graph, vertex, speed=1)

Return the resource footprint tuple for vertex, scaled by speed units per second. The tuple ordering follows raw_materials.

vertex_costs!(costs, graph, vertex, speed=1)
